CUE无法使用以及乱码处理笔记

有时候CUE会无法使用，有的是因为FILE不匹配，用记事本打开后更改FILE后面的那一串为同文件夹下的那个歌曲文件即可。
另一个就是乱码的现象，多为日文或韩文歌曲。主要是编码的问题。
我是这么处理的，后缀改为txt，然后用火狐打开。调整编码直到不乱码(也可以用everedit等能选择编码打开的编辑器)。

有一个奇葩的用繁体中文(BIG5)编码保存的日文cue，在sublime、everedit里会有些字符无法显示。(是这个造成的Unicode补完计画，台湾人对…的开发真是热衷)

当时是别人的电脑，我在XP卸载360，结果系统进不去了。我直接把XP装了个win7，结果会有个旧版本的windows给你选择，一直处理不掉

1、以管理员身份启动cmd窗口【一定要以管理员身份启动，否者提示无法打开启动配置文件，拒绝访问。启动方法可以按照C:/windows/system32/cmd.exe路径找到cmd.exe 然后鼠标右键选择 以管理员身份运行或者在任务管理器里用“运行新任务”】

description Ubuntu 【说明了系统的版本】

3、输入并执行命令bcdedit /delete {f00a81b3-9c18-11de-817f-f75094157968}

bcdedit /displayorder {current}
