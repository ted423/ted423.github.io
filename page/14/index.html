[问题处理]Windows 取消同步中心的启动

نُشر في 2016-08-03 في Issue

问题描述

在访问网络位置的时候，右键点击始终脱机可用会启用同步中心，之后会一直启动，并且无法手动取消同步

解决方案

  1. 在同步中心中找到脱机文件
  2. 进入计算机，找到对应的文件夹，右键取消始终脱机可用的勾选,并删除脱机副本
  3. 找到注册表HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Syncmgr\Handlers
  4. 删除{750fdf10-2a26-11d1-a3ea-080036587f03}这一项后同步中心会消失，并且之前设置的同步才会消失

[问题处理]Lync联系人名片显示异常

نُشر في 2016-08-03 في Issue

症状

Lync是公司内部使用，公司有自己的Exchange服务器。
联系人资料显示异常，如姓名不对(账号相同，更换其他人使用，但仍显示之前的人的姓名)，电话号码不显示，或者其他。

原因分析

由于删除Lync配置无效，但Lync跟outlook有联系，因此需要找到Outlook里相关配置进行删除。

操作

  1. 退出lync, 打开“%appdata%..\Local\Microsoft\Outlook”
  2. 删除“Offline Address Books”文件夹，重新登录Lync
  3. 如仍有问题，在Outlook里找到联系人名片以及Lync联系人名片删除
  4. 如Outlook里删除不了，可能需要登录网页版的邮件服务器里删除，或者联系Exchange服务器管理员

剧情

主角兔朱迪（Judy Hopps）是一只来自农村兔窝镇（Bunnyburrow）的兔子，她在九岁时的一次舞台剧表演上立志要成为一名警察。当朱迪成年后，她进入动物城警官学校学习。大家本以为朱迪不可能当上警察，但她凭著努力，经过艰苦的训练，最终从警官学校毕业并成为动物城警察局的第一位兔子警官。没想到，朱迪离开兔窝镇来到动物城后，报到的第一天却被牛局长（Chief Bogo）指派担任交通警员，负责超时停车开罚单的工作；而在她工作的第一个上午，朱迪还在一间冰店结识了一名骗子狐狸尼克（Nick Wilde）和他的同伙芬尼克（Finnick）。
隔天，朱迪在发现一名叫威斯顿公爵（Duke Weaselton）的黄鼠狼在一间花店行窃后，抛下了交通警员的工作，开始在全城追捕他。尽管她追捕有功，却因她擅离职守且不听上级指示，而遭到了牛局长的责骂，甚至险些被开除。当奥特顿夫人（Mrs. Otterton）来到牛局长的办公室恳求警方帮忙寻找她失踪的丈夫阿密特·奥特顿先生（Mr. Emmitt Otterton）时，在场的朱迪自愿接下了这个案子。在羊副市长（Dawn Bellwether）向狮市长（Leodore Lionheart）发送有关朱迪接下案子的消息后，牛局长被迫地同意她接案，但要求她必须在48小时内破案，否则得交出警徽并辞职。朱迪调阅资料后发现尼克恰是失踪案的关键证人，于是跟踪尼克并以他涉嫌逃税为由逼迫他协助破案。
朱迪和尼克先至位于撒哈拉广场的一家天然主义自然崇拜协会取得阿密特·奥特顿先生的车牌号码后，又到车辆管理局查到了车牌号码对应车辆的注册地址，然后远赴冰川镇的一家礼车出租公司寻找阿密特·奥特顿先生。当晚，他们被冰川镇的犯罪头目大先生（Mr. Big）的手下给带走，而大先生则告诉他们阿密特·奥特顿先生袭击了他的司机曼查斯（Mr.Renato Manchas）。于是他们前往热带雨林区寻找曼查斯，但曼查斯突然发狂，失控地向朱迪和尼克追击；朱迪以无线电呼叫总局协助，后来牛局长和其他警员来到现场，但发狂的曼查斯却无处可寻。牛局长要求朱迪交出警徽，但尼克替朱迪辩护，并坚称他们还剩下10小时可以破案。在离开现场的缆车上，尼克告诉了朱迪他幼年时被草食动物欺负的经历，而该事件也使尼克决意成为恰好符合众人对狐狸刻板印象的狡猾骗子。后来朱迪和尼克在羊副市长的协助下使用道路监视器寻找曼查斯的下落，并循线来到悬崖上的一间废弃医院。他们后来发现，所有失踪的哺乳动物都像曼查斯一样发了狂并被关押在此，且将他们囚禁起来的就是狮市长。后来，到场的警员将狮市长逮捕，而朱迪也因破案有功而一夕出名，成为了英雄。因破案过程中和尼克结下了深厚的友谊，朱迪于是邀请他加入警局成为同伙警员，尼克也高兴地同意了。但在稍后的记者会上，朱迪发言指出那些肉食动物会发狂是和他们的天性有关；此话勾起了尼克幼年被欺侮的回忆，于是他和朱迪争执后愤而离去。往后的一段日子中，仍不断有肉食动物发狂的消息传出，也令动物城的群众开始对身为少数族群的肉食动物产生恐惧和歧视。这样的结果令朱迪认为她并不是英雄，故辞去了警官的职务，回乡协助家人务农。
朱迪回到兔窝镇后，发现用来驱虫的植物“午夜嚎叫”（Nighthowler）能对动物产生严重的精神影响。接着，她驱车赶回动物城寻找尼克并向他道歉；两人和好后一同前往质问先前被朱迪逮捕、在花店中偷窃午夜嚎叫原料、售卖盗版影碟的威斯顿公爵。他们发现，一只名叫道格（Doug）的公羊在地铁站内的秘密实验室里将午夜嚎叫萃取液制成子弹，并射击肉食动物使他们发狂。朱迪和尼克在顺利取得实验室的证据后欲奔往警局，途中却被已成为市长的羊副市长拦截，并发现原来羊副市长才是整个事件的幕后黑手。后来，朱迪和尼克上演戏码欺骗羊副市长，并将她主谋及陷害狮市长的供词录下：她的目的是利用草食动物对肉食动物的恐惧当权，逐出肉食动物并达成她的种族优越主义。后来，赶到现场的警方因证据充足而逮捕了羊副市长。

火狐的autoproxy和Foxyproxy都开始出现这样那样的问题，于是我想到了.pac,没想到的是，这方法简单、有效，不但省了个扩展，还比扩展用起来舒服多了

一个PAC文件包含一个JavaScript形式的函数“FindProxyForURL(url, host)”。这个函数返回一个包含一个或多个访问规则的字符串。用户代理根据这些规则适用一个特定的代理其或者直接访问。当一个代理服务器无法响应的时候，多个访问规则提供了其他的后备访问方法。浏览器在访问其他页面以前，首先访问这个PAC文件。PAC文件中的URL可能是手工配置的，也可能是是通过网页的网络代理自发现协议（Web Proxy Autodiscovery Protocol）自动配置的。

网络代理自发现协议（WPAD）：浏览器通过DHCP和DNS的查询来搜索PAC文件的位置。(与本文无关)

火狐可以打开浏览器控制台进行调试

既定参数

url：the full URL being accessed.
host：the hostname extracted from the URL. it is the exact same string as between :// and the first : or / after that. The port number is not included in this parameter. It can be extracted from the URL when necessary.
ret：(the return value) a string describing the configuration. The format of this string is defined below.

返回值

Instruction Meaning
DIRECT Fetch the object directly from the content HTTP server denoted by its URL
PROXY host:port Fetch the object via the proxy HTTP server at the given location (name and port)
SOCKS host:port Fetch the object via the SOCKS server at the given location (name and port)

SOCKS可以写SOCKS5或者SOCKS4的样子，写SOCKS就是SOCKS4。
区别在于用SOCKS4只支持TCP，而用SOCKS5支持UDP，还有支持各种身份验证机制。但用SOCKS5是远程代理解析DNS(FF是这样)。

WordPress修改固定链接之后,无法访问的解决办法

نُشر في 2016-05-04 في Code

原文链接:http://www.inlojv.com/wordpress-modify-fixed-link.html

主要是方法二所说的问题，也就是apache里mod_rewrite 模块没有启用。

修改apache里面的配置文件httpd.conf

主要修改两个地方

  1. 开启apache中的rewrite模块，在httpd.conf 中找到
    #LoadModule rewrite_module modules/mod_rewrite.so
    把前面的#号去掉，变成
    LoadModule rewrite_module modules/mod_rewrite.so

  2. 开启AllowOverride ，怎么做？ 也是在httpd.conf 中找到AllowOverride None将它改成AllowOverride All
    特别注意的是：AllowOverride可能会有好几处，只需要修改 <Directory /> 和 你站点所在的根目录的一处
    如<Directory “F:/…… /www/htdocs”> 即可
    如果不懂，索性将这几处全部改了 。全部改好，必须保存 并且重启apache 。
    问题即可解决。
