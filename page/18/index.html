[资料]DNS相关

DNS概念
DNS一般分为权威服务节点和递归服务节点。
我们一般使用的都是递归服务节点。

递归DNS(local dns)

递归DNS可以理解为是一种功能复杂些的resolver，其核心功能一个是缓存、一个是递归查询。
收到域名查询请求后其首先看本地缓存是否有记录，如果没有则一级一级向上级递归服务器查询，直到最上级递归服务器开始向根服务器，顶级域名服务器（例如“.com”），一级域名服务器（例如“example.com”）迭代查询……直到获取到结果然后返回给用户。

递归 DNS 由网络运营商建设，提供域名查询解析服务。

递归以及迭代

这里是重点强调下，因为我感觉网上查的不少在这都很混乱
我的理解是local dns，也就是递归DNS的层级可以设置不止一层（最终到根域名服务器）。

客户端和递归DNS之间的查询是递归的。

递归DNS(local DNS)向root name server、顶级域名服务器(TLD DNS),权威DNS(Authoritative name server)的查询是迭代的。

根域名服务器(root name server)

最高级别的域名服务器，全球只有13组根服务器(一组很多台，共用一个IP，而且可以架设不同IP的镜像)。
域名格式为“字母(a-m).root-servers.net”
中国架设了F、I、K、L根镜像服务器。
根域名“.” 服务器，例如www.baidu.com的域名是www.baidu.com.，最后那个“.”通常被隐藏。
通常只有根域名服务器有Root Zone file，而其他递归DNS服务器则只需要配置Root Hits文件，只包含根域名服务器的地址。

顶级 DNS(TLD DNS)

负责处理 .com, .org及所有顶级国家域名，提供到权威域服务器的映射。

权威 DNS(Authoritative name server)

特定域名记录（例如“example.com”）在域名注册商处所设置的 DNS 服务器，用于特定域名本身的管理（增加、删除、修改等）。

权威 DNS 服务器只对自己所拥有的域名进行域名解析，对于不是自己的域名则拒绝访问。比如，向“example.com”的权威 DNS 服务器查询“test.com”的域名肯定会查询失败。

权威DNS即通俗上“这个域名我说了算”的服务器。
权威 DNS 由域名解析服务商建设，提供域名管理服务，维护域名解析记录。

我觉得可以把根服务器和顶级DNS理解为权威 DNS的最高两级

转发DNS

转发DNS是一种特殊的递归。如果本地的缓存记录中没有相应域名结果时，其将查询请求转发给另外一台DNS服务器，由另外一台DNS服务器来完成查询请求。

ECS

用个人看法总结一下，local DNS的查询通常只会提供local DNS的IP，而现代网络为了效能需要由距离用户最近的节点来响应。而ECS是用来附加请求解析的客户端的信息。

[开源]KeePassXC - 强大的密码管理软件以及密码自动填写工具

我在寻找一个在自动填写密码上能超过Firefox自带的扩展，Lastpass算一个，但是他折腾的我连为什么不想用他都忘了，直到我找到了KeePass。

而KeePassXC是KeePass(Windows only)的跨平台版本KeePassX的升级版(瞧这复杂的关系)

KeePassXC

它的优势在官方开发扩展上面，虽说没有KeePass的KeeFox那么强，但绝对够用了，并且FF自宫一刀，KeeFox没得用的现在。基本没有做的更好的。
图标获取功能也是自带的。
KeePassXC的缺点在于其自动输入的标题必须是全字匹配，并且必须单独条目单独设置，不过也因此避免了一些误操作，并且匹配程度更高。
它的License说是 “This program is free software: you can redistribute it and/or modify it under the terms of the GNU General Public License as published by the Free Software Foundation, either version 2 or (at your option) version 3 of the License.”

KeePassXC自身的自动登录

不过注意自动登录有安全风险就是了

KeePass

KeePass是开源软件,采用本地数据库的方式对密码进行管理，软件对密码数据库采用256位AES算法加密，理论上破解难度极大。这样，即使电脑丢失或黑客入侵导致密码数据库被窃取，黑客也很难从中解密出明文帐号密码信息。

KeePass仅凭ctrl+alt+A就能打遍天下无敌手，ctrl+alt+A可以实现“用户名”+tab键+“密码”+enter的输出(甚至你可以自定义)，也就是说无论网页还是游戏窗口亦或是QQ的窗口，你都可以通过ctrl+alt+A自动登录，而且KeePass可以通过识别窗口标题(当前活动)来进行精确匹配。

浏览器扩展以及KeePass插件

其实
通过ChromeIPass(已停止维护)扩展和KeePassHttp插件，可以实现Chrome浏览器填入KeePass数据库密码的功能。

KeeFox则是火狐的扩展，虽然还有passifox,但Keefox更强，passifox连设置项都没有，只负责和KeePass交互。KeeFox则不然，我目前还没见过有页面没法KeeFox填写密码的，甚至在不会提示你保存密码的页面，你也可以在KeePass里手动建立然后用KeeFox填写。并且，KeeFox能记住其他的表单项，不单单只是用户名跟密码，下拉选单例如“登入有效期”之类的都能记住。

相比LastPass简直弱爆了，别说那屁玩意登录都不让我登了，感情我保存在里面的密码我要看还得去求他？不下他那扩展还不让我看？
