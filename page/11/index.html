[不再推荐][大眼仔~旭]Clover 3.5.4 多标签资源管理器，替代Explorer

Veröffentlicht am 2020-05-27 in Software

不再推荐的原始是买了Directory Opus

目前的版本似乎不少杀软都会报毒，但都是报类似Adware或者Softcnapp这类的，请各位自行判断是否使用。

另外注意需要用管理员模式运行才能保存配置

试过不少的多标签资源管理器这方面的软件，初衷就是找窗口方便，下面资源管理不要叠太多免得一个文件夹打开太多次都不知道，然后看起来还乱。

QTTabbar会影响复制粘贴，没办法只在一个窗口显示，有时会打开新的窗口。Groups会卡死。
其他的不少都没法直接替代资源管理，使用替代方案又有这样那样的问题，譬如迅雷会觉得自己没法开文件夹，又重新开个Explorer。

Directory opus是比较好的，但是因为其破解后会有一些报错，本身又有些贵，所以没用。
相较之下，DO更重型，设置项过多，有些复杂，界面调整有些奇怪的逻辑，主页的显示不会以Windows 默认为主，是自己的一套东西，但个人感觉不如Windows的自带的好。

Clover就比较轻型，也有一点小问题就是了，有时鼠标在标签页上移动，虽然没点击但标签页会粘在上面。而且不像DO可以不开重复的路径

已知问题:如果文件夹内有东西会不断变动，Clover会不断刷新，影响焦点，然后影响使用。

http://www.dayanzai.me/clover.html

http://www.carrotchou.blog/26826.html(失效连接)

[Git]一些命令

Veröffentlicht am 2020-04-21 in Code

如无特别说明，本人的所有Git操作都是以Github为主

git 文档
https://git-scm.com/docs

本地分支删除

查看项目的分支们(包括本地和远程)

删除本地分支

reset 删除错误提交的commit

使用这个的前提条件是没有push,一般不建议push -f
git reset --hard <commit_id> 回退到某一版本。
git reset --hard head~1 回退一个版本(数字2自然就是两个版本啦)
然后 git push --force

rebase 伪造历史

git rebase -i HEAD~10 (修改最近10个历史)

下面是rebase打开的文本里命令的提示(git是通过判断进程创建以及进程是否仍然存在来判断用户是否开始、结束编辑，所以多标签的文本编辑器会影响git判断你是否还在编辑)

其中每个历史里的变动占一行，我们只要改前面的pick即可，其中主用的一般是squash，squash会合并改动，再次rebase时是不会拆分的。edit则是修订历史变动里的错误。也可以改动这些变动的顺序。

需要注意的是git里以记录改动为主，如果修改历史变动是会改变现有代码的(特别是你把某个文件的最后一次commit给弄没的时候)。drop 意味着当时进行的修改也会被取消。很容易造成冲突(conflict)，
所以一般是建议squash来合并commit

最后 git push --force
这个命令会把远程的彻底抹掉，出问题也会很彻底的，所以要谨慎。

[不再推荐]Cent Browser - 第三方Chromium修改版

Veröffentlicht am 2020-04-13 in Browser

各版本下载http://static.centbrowser.com/
我尝试过各种浏览器，最终选择的还是Cent Browser。

简单说说为什么吧
傲游: 长期强调自己是webit引擎，拒绝承认自己是Chromium，通过webkit开发出了个网页兼容性跟safari不一样，全方面靠拢Chrome的浏览器，嗯，我信了。
甚至某些GUI上的BUG还跟Chrome保持同步。当时反馈到论坛，结果人家只会复制粘贴糊弄，反倒是金山猎豹，耐心解决了这个Bug（也可能是更新了版本。

一些国产浏览器（如果不流氓的话还能接受

360极速，有了Chrome没有的Bug，明明完整的复现了，结果论坛上人家懒得看，直接启动糊弄模式。

原生Chrome:书签栏没法双列，使得某文件夹大量收藏的话会很难找东西，很多扩展没法修改的功能没有，比如复制连接文字，收藏的替换为当前网址，还有重中之重的调用下载器。
我知道不少人要提aria2c，但实际上YAAW一直开启的话还会影响到一些其他功能，充其量这只是一种不得已的替代实现。

FF:最早我是FF死忠来的，甚至自己改扩展，改脚本，直到我试图翻译文章被封。(目测短期操作太频繁了)
FF完成了一次涅槃，但很难讲这种变化是好的，大量优质扩展直接暴死，现在的扩展基本和Chrome差不多。然后FF仍然以程序员思维在做这款浏览器，一些奇奇怪怪的Bug都是减分项。(下载东西还能下download.php下来)
如果以前的FF以强大的可自定义性为特点的话，现在的FF真不知道还剩下什么了。如果你什么都学Chrome，那我为什么不用Chrome呢？
