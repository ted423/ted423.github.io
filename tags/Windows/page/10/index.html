本地同步软件

目前自动的同步应该算全军覆没了。

FreeFileSync

http://www.freefilesync.org/
说是同步，但最主要的功能还是实现两个文件夹的对比。能多文件夹进行对比，并且自定义之后的同步规则，例如删除还是覆盖或是复制到右侧
应该是通过大小来进行比对的，但如果文件有损坏，该软件无法发现
不能识别移动或者改名之类的操作，比如将文件夹内的文件改名，软件只是单纯的判断文件有无然后进行删除和拷贝。不过由于判断机制比较简单，不大容易出错。

Beyond Compare

混入了一个奇怪的东西
相比上面那个更完善，除了收费以外没啥问题

局域网内同步 自同步
http://www.zisync.com/

特点是不需要服务器，能自动同步不出错
缺点是对于大文件(大过20G的)，无法完成同步，并且已停止更新（2015年最后一次）

Resilio

https://www.resilio.com/individuals/
之前之所以不提是因为这个软件会涉及到中间的服务器，如果是公司内部用其实是有一定隐患的，特别是一些对内部管控比较严的公司。后来发现是可以不选用中继服务器跟跟踪服务器，只要指定预定义主机就好。

这个软件主要做局域网同步，本机的文件夹同步就没办法了，比方说同步移动硬盘上的文件夹之类的。另外就是会生成一个.sync的隐藏文件夹，用来存放删除的文件。
注意这个不能在服务器上使用，需要花钱

SyncTrayzor/Syncthing

印象里不太好用，不过基本功能还是有的，待评测
比较推荐，毕竟开源
https://github.com/canton7/SyncTrayzor
这个是Windows GUI，MIT

又不推荐了，因为这个 https://forum.syncthing.net/t/directory-deletion-on-remote-computer/15941/31，使用到现在，小问题不断。
单纯做一个仅接收的同步逻辑都会出问题，那么说明这个软件同步完全不行。
我倾向于认为删除整个父文件夹，对 Syncthing 来说没有删除子文件的 Index导致的。毕竟源目录整个目录都没了。被同步的目录还在报目标路径还有文件就很奇怪

  • https://syncthing.net/
  • https://github.com/syncthing/syncthing

后者是核心进程，MPL-2.0
不过我也还没在Linux上用过

RAMMap - 内存分析工具

https://docs.microsoft.com/zh-cn/sysinternals/downloads/rammap

其中显示项目的说明

来源于:http://blog.darkthread.net/post-2013-10-27-where-have-all-the-memory-gone.aspx

微软文档: https://techcommunity.microsoft.com/t5/Ask-The-Performance-Team/Introduction-to-the-new-Sysinternals-tool-RAMMap/ba-p/374717

  • Process Private: 分配给每个进程专用的内存
  • Mapped File: 用来储放文件内容缓存(Cache)的内存空间
  • Shared Memory: 标注给多个进程共用的内存分页(Page，内存管理单位)
  • Page Table: 用来描述虚拟内存地址的分页表(里面是一笔一笔的PTE，Page Table Entries)
  • Paged Pool: 允许移至磁盘的核心集区内存(Kernal Pool Memory)
  • Nonpaged Pool: 不允许移至磁盘的核心集区内存
  • System PTEs: 与I/O空间、核心堆栈、内存描述清单等系统分页相关的PTE
  • Session Private: 登录工作阶段相关的内存
  • Metafile: 是系统缓存的一部份，包含NTFS Metadata(包含MFT及其他NTFS Metadata文件)。在MFT中，每个文件属性记录占用1K，而一个文件至少有一个属性记录，再加上其他NTFS Metadata档，当文件数众多，这块会很快速成长。
  • AWE: 启用Address Windowing Extension技术所使用的相关内存空间(较常应用在SQL或其他DB)
  • Driver Locked: 驱动程序锁定的物理内存。多用于I/O的暂时性小量应用，如果有装RAMDisk，也会算在这一区。
  • Kernel Stack: 核心线程推叠，线程愈多，用量愈大。

每项分类都有以下字段:

  • Active: 正在使用中的物理内存分页(Process Working Set或System Working Set)
  • Standby: 留在物理内存但暂不使用的分页，保留供后续能快速重复利用
  • Modified: 与Standy类似，但内容被修改过，重复使用前要先回写到磁盘机
  • Modified no write: 与Modified类似，但标注为不需回写到磁盘
  • Transition: 在分类之间转换的分页
  • Zeroed: 内容已清空可供使用的分页，系统刚开机时明显增加，随着使用一段时间逐步转为Standby
  • Free: 可以使用但残留先前数据的分页，使用前需先转为Zeroed
  • Bad: 标注损坏的内存

[软件推荐]有用的小工具们

批量重命名软件类

https://ted423.github.io/Software/%E6%89%B9%E9%87%8F%E9%87%8D%E5%91%BD%E5%90%8D%E8%BD%AF%E4%BB%B6/

Attribute Changer

同File Property Edit一样可以进行文件属性修改，但主要是通过右键菜单，
http://www.petges.lu/home/download/

File Property Edit

文件属性修改,你可以将普通文件改成系统文件，十分适合用来藏东西
http://www.foryoursoft.com/download.htm

搜索类

DocFetcher

开源(EPL)的文件内容搜索，不止支持文本文件，还支持doc以及pdf等文档
不过需要进行索引
最重大的缺点就是不能搜索连续的符号，但有时候需要精准结果的时候，这个软件靠不住
http://docfetcher.sourceforge.net/en/index.html

Everything

搜索文件用，相当出名的文件搜索工具，会通过主文件表中的文件元数据创建卷上每个文件和目录名称的索引，速度相当快，缺点就是只支持文件名，不支持附加条件的搜索，例如文件大小和创建日期
http://www.voidtools.com/

FileSeek

这个软件是专门为了搜索文件内容而生的。没有locate32的小毛病，后面感觉用FileLocator会好些，因为我发现FileLocator添加默认为文本的文件类型会比较方便。我印象里FileSeek最早也是直接添加的，不知道是我记错了还是后面改了。
https://www.fileseek.ca/

FileLocator

跟上面算是同质同类的软件
https://www.mythicsoft.com/filelocatorpro/

locate32

BSD License
个人目前没在用，因为FileSeek或者Docfetcher明显更好用
搜索文件用,虽然相对于Everything来说不是那么快速，但是功能更为强大，可以搜索文件大小、日期等等
就是文件内容搜索是将DBCS与UTF-8分开的，搜索utf-8文本文件的话，需要以utf8:开头。
长期没有更新了
https://sourceforge.net/projects/locate32/

Copyq

GPLv3
用就是了 之前在用Ditto，不过感觉Ditto小bug有点多
https://github.com/hluk/CopyQ

Exeinfo PE

放在Github上的闭源软件,之前有一堆不同域名的主页
https://ted423.github.io/Software/Exeinfo%20PE/

HashCheck

Standard 3-Clause BSD License.
https://github.com/gurnec/HashCheck
因为Robin Keir的hash在特殊字符上有些问题，找替代软件找到的。不过只能在右键属性里查看。如进行文件对比，操作量会略微增加。

MyHash

Robin Keir的hash的完全替代品,可惜虽然在Github上，但没有开源。
https://github.com/drag0n-app/MyHash

KeePassXC

强大的密码管理软件以及密码自动填写工具
https://ted423.bitcron.com/post/software/keepassxc

微软小工具合集

微软推荐的系统工具合集
https://ted423.bitcron.com/post/software/xi-tong-gong-ju/wei-ruan-xiao-gong-ju-ji-he

Registry Workshop

注册表搜索工具,搜索速度快，可以批量操作,不过需要付费
https://hrtsea.com/2133.html
官网：http://www.torchsoft.com/

RightMenuMgr

右键菜单管理，最详尽最专业的，官网已消失，虽然是老软件但在win10下也正常。
https://www.lanzous.com/i39jpri

SpaceSniffer

是一款非常直观的磁盘空间占用查看应用，还支持根据不同类型文件进行过滤显示。后面感觉WizTree更好
http://www.uderzo.it/main_products/space_sniffer/

WizTree

比较而言后者大文件提示会比较直观.百分比占用会比图形更准确，好理解。
前者会边扫图形边变化，前期图形会失真，并且整体速度略慢
据说Wiztree是直接对NTFS文件系统的MFT进行文件分析，不过我测试是对的网络路径(小文件太多估计有影响)
固此，我倾向于用WizTree 完全取代 SpaceSniffer
https://www.diskanalyzer.com/

SUMo

软件升级检测，最神奇的是他会跟据快捷方式来检测软件，还可以自己添加检测文件夹。不单单是从系统的程序和功能里检测，也就是一些绿色软件甚至单exe的小工具也能检测到。
其中扫描就是确定有什么软件以及当前软件版本，检查就是去服务器查最新版本。
其实这个公司还有出一个检测驱动的叫DUMo,不过会有乱码并且Win10自己有升级驱动的机制了。
http://www.kcsoftwares.com/?sumo

UnLock IT

文件解锁，删除删不掉的文件都可以用
各种解锁软件在我上Win10就开始出问题，比如IObit Unlocker,又比如LockHunter然后我找到了UnLock IT，这是一个免费软件,功能强大。
http://emcosoftware.com/unlock-it/download
