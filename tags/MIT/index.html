[开源]软件更新管理 - Winget

Winget 打败了所有（
所以地址由 Software Update 改成了 Winget

Winget

开源，MIT，开源地址 https://github.com/microsoft/winget-cli
Winget 永远的神，Win 10 开始自带
目前除了批量升级有点使用上的问题以外，其他做的都挺强

目前可以通过以下方式批量升级(也可以用来备份恢复软件列表)

  1. winget export -o 文件
  2. 删除列表里不需要的文件
  3. winget import 文件

winget 下的升级文件都在 %Temp% 里，一些软件升级时需要关闭其他软件可能会导致升级失败，可以手动运行安装包升级

https://winstall.app/apps

这里可以自己选软件然后生成批量安装的代码

WingetUI

开源 LGPLv2.1

WingetUI 其实不依赖系统的 winget,他自己又带了一个，系统的是appx，然后配置放在软件的同目录下，而 WingetUI 的 winget 的配置是放在 C:\Users\XX\AppData\Local\Microsoft\WinGet — 2023-07-18

https://www.marticliment.com/wingetui/
https://github.com/marticliment/WingetUI
第三方UI，目前没弄懂为啥他获取的会比命令行的要新
不过有忽略模式就行。

首先，下面两个软件主要是替代国产软件管理用的，并且属于一种不成熟的替代品。
如果你想寻找一种更好的体验，那大概是可以放弃看这篇文章。

Sumo

http://www.kcsoftwares.com/?home
夸一夸Sumo，真的很强大，他的软件检测是基于快捷方式，也就是便携版也支持检测。这点已经比360等的强了。
不过不给钱他只是一个检测器而已，甚至连从开发者网站获取更新都不让你用。

这个软件商还有个产品Dumo是升驱动的，不过Win已经大包大揽，再不济还有各类厂商的自用检测。(有点Out of topic)

Chocolatey

Chocolatey 虽然是开源的，但其实际还是一款面向企业的商业产品，并且国内体验不好。Apache License 2.0
像什么微信QQ企业微信，里面是没有的。
免费产品限制了功能，并且只能装到C盘。之前还有安装失败也会计入软件列表的问题

因此这个产品的主要定位是一定程度上处理软件升级的问题。主要处理软件自带升级智障的问题，例如WinSCP(之前有用scoop，这个自己本身也智障，配置放便携版里，结果一更新配置没了)

GUI

choco install chocolateygui

注意的点

  • 这软件在Win7上要更新PowerShell。
  • 只有想装C盘并且不需要安装包配置关联的软件才用Chocolatey直接装，否则建议自行安装，其后用Chocolatey升级
  • 社区维护，因此很多软件安装行为都由社区维护者决定
  • 因为上面一条，可能面临没有维护或者不再维护的情况
  • 网络问题
  • 还可能有升级报错的情况，如OBS-Studio
  • 部分时候需要重装.install https://github.com/chocolatey/choco/issues/1027
  • 他的软件列表和计算机的列表是相互独立的，并且不能单纯只删记录(只能pin)

一些常用命令

  • 列出本地安装软件 choco list --local-only
  • 列出可升级的软件 choco outdated
  • 安装指定等级 choco install virtualbox --version 4.3.26
  • 不检查更新choco pin add -n=googlechrome
  • choco search 名字
  • Proxy Server (不支持socks5,)

然后他们之间打了一架，360弹窗给用户洗脑，往傲游和金山网盾身上扔了一堆屎盆子。

本地同步软件

目前自动的同步应该算全军覆没了。

FreeFileSync

http://www.freefilesync.org/
说是同步，但最主要的功能还是实现两个文件夹的对比。能多文件夹进行对比，并且自定义之后的同步规则，例如删除还是覆盖或是复制到右侧
应该是通过大小来进行比对的，但如果文件有损坏，该软件无法发现
不能识别移动或者改名之类的操作，比如将文件夹内的文件改名，软件只是单纯的判断文件有无然后进行删除和拷贝。不过由于判断机制比较简单，不大容易出错。

Beyond Compare

混入了一个奇怪的东西
相比上面那个更完善，除了收费以外没啥问题

局域网内同步 自同步
http://www.zisync.com/

特点是不需要服务器，能自动同步不出错
缺点是对于大文件(大过20G的)，无法完成同步，并且已停止更新（2015年最后一次）

Resilio

https://www.resilio.com/individuals/
之前之所以不提是因为这个软件会涉及到中间的服务器，如果是公司内部用其实是有一定隐患的，特别是一些对内部管控比较严的公司。后来发现是可以不选用中继服务器跟跟踪服务器，只要指定预定义主机就好。

这个软件主要做局域网同步，本机的文件夹同步就没办法了，比方说同步移动硬盘上的文件夹之类的。另外就是会生成一个.sync的隐藏文件夹，用来存放删除的文件。
注意这个不能在服务器上使用，需要花钱

SyncTrayzor/Syncthing

印象里不太好用，不过基本功能还是有的，待评测
比较推荐，毕竟开源
https://github.com/canton7/SyncTrayzor
这个是Windows GUI，MIT

又不推荐了，因为这个 https://forum.syncthing.net/t/directory-deletion-on-remote-computer/15941/31，使用到现在，小问题不断。
单纯做一个仅接收的同步逻辑都会出问题，那么说明这个软件同步完全不行。
我倾向于认为删除整个父文件夹，对 Syncthing 来说没有删除子文件的 Index导致的。毕竟源目录整个目录都没了。被同步的目录还在报目标路径还有文件就很奇怪

  • https://syncthing.net/
  • https://github.com/syncthing/syncthing

后者是核心进程，MPL-2.0
不过我也还没在Linux上用过

[软件推荐]图片类相关软件

看图

最近没怎么看漫画就比较尴尬，目前还是Honeyview，能用就行，不过感觉他的缩放算法问题很大

我一度是想抛弃ACDsee的，比如文件名长度问题导致的崩溃，又比如某些gif会花。但偏偏他的数据库让人无法舍弃，比方说按宽高搜索图片之类的，速度相当快，相对来讲XnViewMP搜索类别更细，功能性更强，但是Xnview里的结果是一次性的，一旦点开一张图片结果就会消失。就个人感觉，XnView的规划很乱，设计也是乱七八糟，唯有功能比较多而已。
况且XnView和XnViewMP的使用界面和方式还有很多不一样，感觉怪怪的。

因为Honeyview说除安全修补，不再更新，所以想找替代，结果Honeyview 2021还有更新，nomacs一年没更新(现在都在action里下最新版)，FastStone MaxView两年。
