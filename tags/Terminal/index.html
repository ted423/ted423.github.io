Windows 路由相关

多网络环境可以进行路由优先级调整，避免类似不能访问外网的路由优先级高导致电脑不能上网
其实调整跃点就可以
对于内部的，不跨网段的，其实可以把网关删掉

希望优先级低的，在适配器处，进入ipv4设置，然后点击高级，取消自动跃点，给跃点数填个很高的值

route print
可以看到路由优先级

可以手动指定路由
route add 192.168.0.0 mask 255.255.255.0 192.168.68.254

-p 代表是永久路由

在 Linux 下如何强制取消挂载(umount) NFS 目录

这种情况下可以考虑lazy umount
即 umount -lf 路径
lazy umount是针对busy而提出的，即可以卸载“busy”的文件系统。
请注意，该方法并不是完全安全的

另外可以参考下https://blog.miniasp.com/post/2008/10/20/How-to-unmount-NFS-directory-under-Linux-when-hanging

[Terminal]Windiows系统下删除启动选择系统项

感觉这个文章有点过时了 –2019-06-11
首先vista 及以后系统是不用boot.ini的，而msconfig无法删除。(老实说很早以前的事了，具体记不清也懒得再现环境)

当时是别人的电脑，我在XP卸载360，结果系统进不去了。我直接把XP装了个win7，结果会有个旧版本的windows给你选择，一直处理不掉

1、以管理员身份启动cmd窗口【一定要以管理员身份启动，否者提示无法打开启动配置文件，拒绝访问。启动方法可以按照C:/windows/system32/cmd.exe路径找到cmd.exe 然后鼠标右键选择 以管理员身份运行或者在任务管理器里用“运行新任务”】
2、输入并执行命令bcdedit，得到如下结果：

description Ubuntu 【说明了系统的版本】

其中最后一条记录正是需要删除掉的ubuntu，它的ID号是{f00a81b8-9c18-11de-817f-f75094157968}

3、输入并执行命令bcdedit /delete {f00a81b3-9c18-11de-817f-f75094157968}

bcdedit /displayorder {current}
不显示选择 直接启动windows7

MobaXterm

一直想找一个SecureCRT的开源替代品，Tera Term做的不错。
但是其断开自动关闭的傻逼特性实在让人受不了。
虽然我试图用自动记录log来弥补这个问题，但替代手段终究是替代手段。
想想看，你正在跑的脚本因为一时连接不畅断了，你还要翻log找他跑到哪了。
然后保存telnet密码和SSH密码还得用不同的步骤折腾。
所以MobaXterm成了最终解。

另一方面，不要过度迷信开源，毕竟在处理需求上面，商业软件做的更好。

https://mobaxterm.mobatek.net/

缺点

SFTP问题比较大，经常上传下载文件夹会没法保存文件夹结构，所以建议带文件夹结构的还是用 Filezilla 或者 WinSCP
不过这个很多时候是 ssh 断连的问题，因为 sftp 和 shell 是两个独立的 ssh 连接 - 2023-05-03

破解

https://github.com/flygon2018/MobaXterm-keygen
