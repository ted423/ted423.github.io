bcdedit /displayorder {current}

[问题处理]此程序(应用)正在阻止windows关机(重新启动) 导致无法自动关机的处理办法

这个问题困扰了我好久，从windows 8.1 U1 退回到windows 7 结果仍旧关不了机，必须得点那一下。笔记本就没这种问题。一开始以为是笔记本上曾经的某优化软件帮了我，但在台式上装了金山卫士也无效(现在想想,可能是Windows优化大师)

个人处理方法

打开注册表管理器，找到

新建一个AutoEndTasks，值改为1
注意新建的时候选字符串值，也就是类型那里是REG_SZ

找到WaitToKillAppTimeout(找不到就新建，类型应该是同上)
数值改小即可

不过笔记本的这个值没改过，我想大概是有软件直接取消了阻止Windows 关机这个功能吧。

目前看来，应该是两个值需要同时设置才行(Windows 10 1709) –2018-02-26

Windows Update 以后会偷偷再把值给取消掉 –2019-07-03

“时间提供程序 NtpClient: 在尝试联系域控制器 X 8 次以后，仍然没有收到有效的响应。此域控制器将不再作为时间源使用，并且 NtpClient将尝试发现一个新的域控制器来同步。错误为: 无法连接到对等机。”

[问题处理]Windows 10 1903处理 下载 变成 Downloads 的问题

首先网上查了教程，基本都说是desktop.ini丢失的问题,但我遇到的问题不完全是这个问题。

起因

原因大致是我把Downloads文件夹挪到了另一个盘又挪回去，然后一堆中文里就夹了一个英文的Downloads，像我这种强迫症根本受不了。

推测

这个注册表下正常的应该有

但现在已经消失了(实际上我处理完后，甚至包括下载另一个语言包，在本账户下都没有这一项,也就是处理方法并不能完全解决问题，不过起码不影响强迫症了)
如果手动建立，删除Downloads文件夹，那个项目又会消失
单纯的从别处弄desktop.ini或者新建并不产生任何影响

处理办法

删掉Downloads文件夹，新建一个账户，登进去再登回原来账号，找到新用户目录下的下载文件夹，直接剪切，移到现在的相应位置，再重启可以看到就正常了。

继续推测

Windows 10 现在语言包也通过Windos Store来打，所以现在影响这些不单单通过desktop.ini和注册表来显示中文
