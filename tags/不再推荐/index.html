[不再推荐][截图]ShareX

第二次不再推荐 - 2024-07-15

跑去用 fastone capture 了

  • 设置项过于复杂，设置分到了各个地方，却没有一个明确的分类的方法。找起来十分的累，没什么用的选项过多
  • 滚动截屏印象里存在问题
  • 文件名命名规则虽然设置了但部分情况下不起作用，估计是被他那过于复杂的设置系统拖累了

最后一点是最后的稻草

不再推荐的原因如下
新装(已经是第二次装了)之后各种按键魔幻的要死，然后自动给你上传，然后自动报错。
虽然之前的机子设置过，但第二次找设置来回翻了两三遍
个人感觉这个软件设计其实不太合理，一个软件的设置项过多其实是开源软件的老毛病了。就个人想法，软件是作为工具存在的，那么应该是节省时间而不是让人花费一大堆不必要的精力去适应。

目前新装第一次截图已经有提醒是否进行上传了，取消后正常，说明有人反馈并被接受。
目前仍有设置项复杂的情况，一级带单里就有任务设置，应用程序设置，和快捷键设置。

当然，使用的理解上可能也有翻译带来的问题，例如截图后里面可以设置是否上传。

总之因为开源软件是可以所有人参与贡献的项目，姑且我认为这个软件是未来可期的，故收回之前的不推荐(虽然不推荐那些观点都没发布就已经被收回了，不过曾经确实曾经想过)

  • https://getsharex.com/
  • https://github.com/ShareX/ShareX

Greenshot长期不更新，这个开源带长截图，装FFmpeg后支持录屏以及录gif。
虽然1.3.4之后一直没版本，但是一直有新得commit

[不再推荐]Slax - Linux PE

不推荐的理由在于Ubuntu和Manjaro的安装盘就能做到，并且一般效果比这个还要好
2年未维护，似乎没有提供源码

其实要有个WinPE能看Linux的文件系统也就不用这玩意了
https://www.slax.org

但目前来讲这东西还算有用。
修复了一个ESXi的文件损坏的问题

请注意这个iso不是像别的一样拿什么东西写到移动硬盘或是U盘上，而是直接解压（NTFS文件系统都行）,移动硬盘或U盘中可以保留slax这个文件夹（直接放进去），然后跑boot里的bat。

另外注意这个对移动硬盘可能相性不好，也可能是文件系统的锅，建议优先U盘

[不再推荐][大眼仔~旭]Clover 3.5.4 多标签资源管理器，替代Explorer

不再推荐的原始是买了Directory Opus

目前的版本似乎不少杀软都会报毒，但都是报类似Adware或者Softcnapp这类的，请各位自行判断是否使用。

另外注意需要用管理员模式运行才能保存配置

试过不少的多标签资源管理器这方面的软件，初衷就是找窗口方便，下面资源管理不要叠太多免得一个文件夹打开太多次都不知道，然后看起来还乱。

QTTabbar会影响复制粘贴，没办法只在一个窗口显示，有时会打开新的窗口。Groups会卡死。
其他的不少都没法直接替代资源管理，使用替代方案又有这样那样的问题，譬如迅雷会觉得自己没法开文件夹，又重新开个Explorer。

Directory opus是比较好的，但是因为其破解后会有一些报错，本身又有些贵，所以没用。
相较之下，DO更重型，设置项过多，有些复杂，界面调整有些奇怪的逻辑，主页的显示不会以Windows 默认为主，是自己的一套东西，但个人感觉不如Windows的自带的好。

Clover就比较轻型，也有一点小问题就是了，有时鼠标在标签页上移动，虽然没点击但标签页会粘在上面。而且不像DO可以不开重复的路径

已知问题:如果文件夹内有东西会不断变动，Clover会不断刷新，影响焦点，然后影响使用。

http://www.dayanzai.me/clover.html

另外胡萝卜周的

简单说说为什么吧
傲游: 长期强调自己是webit引擎，拒绝承认自己是Chromium，通过webkit开发出了个网页兼容性跟safari不一样，全方面靠拢Chrome的浏览器，嗯，我信了。
甚至某些GUI上的BUG还跟Chrome保持同步。当时反馈到论坛，结果人家只会复制粘贴糊弄，反倒是金山猎豹，耐心解决了这个Bug（也可能是更新了版本。

一些国产浏览器（如果不流氓的话还能接受

360极速，有了Chrome没有的Bug，明明完整的复现了，结果论坛上人家懒得看，直接启动糊弄模式。

原生Chrome:书签栏没法双列，使得某文件夹大量收藏的话会很难找东西，很多扩展没法修改的功能没有，比如复制连接文字，收藏的替换为当前网址，还有重中之重的调用下载器。

如果以前的FF以强大的可自定义性为特点的话，现在的FF真不知道还剩下什么了。如果你什么都学Chrome，那我为什么不用Chrome呢？

[不再推荐]ultracopier - 复制加强工具

目前不建议使用
我测试了2.2.0.5的版本，目前BUG还有点多，不建议日常使用
会有一些Bug,比如回收站里往外拷东西会导致错误的名字，或者权限问题造成无法拷贝的问题。
所以建议一般使用不要开代替，有需要再开
PS: Fastcopy是真的慢，小文件一多连自带的都比过了，大文件的话又完全不需要用

https://ultracopier.first-world.info/
这是我主动寻找的工具，平时在拷贝文件夹的时候，总觉得慢的难以忍受。究其原因，主要是因为小文件太多。
当时就想找一款多线程的复制工具，这样就可以尽量保证跑满硬盘的读写速度。
最初是找到了robocopy,微软自带的工具。它也成功的在复制时把硬盘速度跑满了。
遗憾的是，作为命令行工具，日常操作中，在你打命令行的时候，很多复制都能完成的差不多了。并且蛋疼的是，对一些无法拷贝或者重名文件，命令行工具就是一场灾难。
如果出错了重新复制，他还需要去花时间对比。

然后去找了一些GUI，但实际的效果可能还不如用命令行快。

其中也有试过fastcopy，但感觉实际上这还是单线程复制。CopyHandler（应该是支持多线程，不过没调设置还不如fastcopy,调了之后崩溃了，没再试），做了些复杂操作，进程崩溃了，复制到一半的东西像正常一样显示在硬盘中，试想一下，以后你都搞不清楚文件是不是正常的。

之后找到了ultracopier(开发者告诉我Supercopier 4已经停止支援)，试了下文件夹移动，比fastcopy快了一倍。从软件行为看（用everything看，先把整个目录层次都copy了过去，然后文件创建非常快），应该是多线程。

需要注意的是这个软件是会替换掉微软自带的复制以及移动行为，因此在回收站里操作会有一些问题。还有设置项里performence有个block size,我是调成了4096KB，不然默认的速度会慢，不明白为什么，感觉inode thread是线程的意思，调高似乎会好些

购买专业版是针对CPU有优化的，个人感觉差异不会太大吧？

目前使用发现了一些BUG

  1. 从回收站往外移动时，会把原本的名字丢掉(Windows回收站对名字的特殊处理导致的)
  2. 往exFAT上放文件会容易出问题，经常一取消整个电脑就卡死了
  3. 网络路径也会有些问题，比如网络不稳定的时候
  4. 没有权限的文件夹会比较尴尬，Fastcopy是有提供一个申请管理员权限的按钮
  5. 还有就是剪切时可以粘贴两次到同一目录，第二次点覆盖会导致文件直接丢失
  6. 不支持获取管理员权限，如果一直以管理员权限运行会有些隐患
