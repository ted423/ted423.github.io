网络同步盘缺陷记录

优秀

  • dropbox
    同步不够实时(服务器端的改变并不会立刻同步回本地)
    如果你希望他立刻进行一次完全的同步，你需要退出他的客户端再重新打开。
  • Onedrive
    不支持局域网内同步,同步逻辑有缺陷，偶尔会干删一堆文件到回收站再重新下载的事(由于下调存储空间一事，已经不准备继续使用，准备转yandisk看看，15元/月才50G，我就呵呵了)
  • 快盘
    国内做的最好的同步盘，不过服务器实在…
    客户端弹窗简直烦人
    因为有秒传机制，会出现上传的某些文件永远无法下载（你也没法重传，因为秒传了）
    因为上一条，是否使用自己斟酌

可用

  • mega
    支持创建同名文件(夹)，不利于同步，同步依据文件修改时间，在修改时间不变改变文件的情况下会去下载源文件覆盖修改后的文件。
    在其他电脑上直接在网页端上传文件同步回本地后会再次进行上传（可能跟装了浏览器扩展有关），然后创建出同名文件夹(我这里甚至出现了重启次机子就会重新上传的状况)
  • Yandex
    同步时会导致同步文件夹内的文件写入慢，影响其他读取该文件夹的软件运行
    不支持回复文件历史
    * 新浪微盘同步盘
    同步时暴力拒绝其他修改，你可以把他理解为一个手动同步工具…
    需要修改时你就暂停他的同步吧。
    * 酷盘(已死)

不推荐使用

  • Google Drive
    这同步简直败坏谷歌的名声（虽说谷歌做的所有软件一直不是我喜欢的，因为感觉太偏执了）
    支持创建同名文件(夹)，不会进行匹配，所以多端同步做的简直shit,你只能让他自己去同步。如果你自己往里放文件协助同步你会发现服务端建立了一堆同名文件或文件夹，然后你会在另一端看到一堆wrong,逗我。
  • 360同步
    只支持单任务，在上传时居然会不去判断网盘内的修改，然后就出现了完全重复的两个文件。基本只是个玩具，根本算不上同步盘。如果你希望他把你的资料弄得乱七八糟，你可以使用他。
  • 百度云同步
    服务器相当不稳定，一般的同步问题都是服务器导致的，甚至有时明明同步未完成，网盘也会判断同步已完成。百度对部分视频进行了破坏性修改，突破了网盘的底线，因此不建议使用这个网盘存储自己的资料
  • 微云
    我当时用的时候，微云是文件名同步，即便改变文件内容，微云也不会去重新同步

下载、资源类软件缺陷记录

BUG

  • aria2c https://ted423.bitcron.com/post/software/zi-yuan-lei/aria2c
  • Transmission
    1. tracker连接性不强，获取tracker信息过慢
    2. 制作种子不支持网络映射
    3. 单线程，牺牲性能降低占用(理论我觉得rt会更好，不过rt不支持改名文件夹这些，并且两个长期不更新的软件，tr反倒是最后更新了一下)
    4. 部分种子识别有问题，部分种子无法使用(我提issue被ban)
  • qbittorrent
  1. 制作的种子dmhy无法读取中文文件名(但应该是网站问题)
  2. 制作的种子tracker中必须隔开一行，否则traker其他软件无法识别
  3. Others https://github.com/qbittorrent/qBittorrent/issues/created_by/ted423
  • 迅雷
  1. 做种相当week,特别是点击添加重新下载时会直接把已下载文件从磁盘抹掉
  2. 现在的版本不能从别的地方下好然后拉进去补充进度了
  3. 迅雷并不会显示具体的tracker信息，并且也不会具体的显示谁给你传数据，也不会显示你在给谁传数据，个人感觉它是努力淡化BT原理，营造一种速度都是迅雷提供的错觉，所以很多迅雷用户体现一种反智现象
  • Deluge
  1. 对中文支持不好,添加种子时路径有中文就出错了，但BT下载的文件夹又可以是中文
  2. 2.0.3大概是不准备也没法支持Windows了

光盘映像类软件

#Rufus
https://rufus.ie/
为了找个替代UltraISO的软件特意去寻找的。开源、GPLv3。
就做启动盘来说，软件非常专业,甚至Ultraiso无法识别的硬盘盒，这个软件也能正常工作。
感觉理论上也可以取代Win32 Disk Imager(还没试装Linux正不正常)
缺点是只支持制作启动盘(不过这种只要直接解压也就行了)

有点奇葩的是，默认的和便携版其实是完完全全一模一样的程序。
没有任何区别

#cdrtfe
这个东西看起来是刻光盘用的，开源，但我整不明白。

#UltraISO
UltraISO 是一个运行在Microsoft Windows平台上的用来创建、修改和转换ISO文件的软件。
当然啦，最主要用来创建U盘启动盘装系统。
官网:http://www.ezbsystems.com/ultraiso/index.html

#Win32 Disk Imager#
必须得说，Linux类的制作启动盘这个可能更好一些。比方说CentOS的，UltraISO写到U盘里用不了。
缺点是如果U盘不再用Ultraiso写一遍的话(先用别的分区工具重新建分区)，Windows下，会无法在磁盘管理修改和分配盘符(也可能可以直接用其他分区工具重建MBR，忘了试了)
ubuntu 15用UltraISO就无法正常启动
http://sourceforge.net/projects/win32diskimager/

#ImDisk Toolkit
能挂载各种奇奇怪怪的东西,可以用来挂win挂不了的东西。
https://sourceforge.net/projects/imdisk-toolkit/

FreeFileSync

http://www.freefilesync.org/
说是同步，但最主要的功能还是实现两个文件夹的对比。能多文件夹进行对比，并且自定义之后的同步规则，例如删除还是覆盖或是复制到右侧
应该是通过大小来进行比对的，但如果文件有损坏，该软件无法发现
不能识别移动或者改名之类的操作，比如将文件夹内的文件改名，软件只是单纯的判断文件有无然后进行删除和拷贝。不过由于判断机制比较简单，不大容易出错。

Beyond Compare

混入了一个奇怪的东西
相比上面那个更完善，除了收费以外没啥问题

局域网内同步 自同步
http://www.zisync.com/

特点是不需要服务器，能自动同步不出错
缺点是对于大文件(大过20G的)，无法完成同步，并且已停止更新（2015年最后一次）

Resilio

https://www.resilio.com/individuals/
之前之所以不提是因为这个软件会涉及到中间的服务器，如果是公司内部用其实是有一定隐患的，特别是一些对内部管控比较严的公司。后来发现是可以不选用中继服务器跟跟踪服务器，只要指定预定义主机就好。

这个软件主要做局域网同步，本机的文件夹同步就没办法了，比方说同步移动硬盘上的文件夹之类的。另外就是会生成一个.sync的隐藏文件夹，用来存放删除的文件。
注意这个不能在服务器上使用，需要花钱

SyncTrayzor/Syncthing

印象里不太好用，不过基本功能还是有的，待评测
比较推荐，毕竟开源
https://github.com/canton7/SyncTrayzor
这个是Windows GUI，MIT

又不推荐了，因为这个 https://forum.syncthing.net/t/directory-deletion-on-remote-computer/15941/31，使用到现在，小问题不断。
单纯做一个仅接收的同步逻辑都会出问题，那么说明这个软件同步完全不行。
我倾向于认为删除整个父文件夹，对 Syncthing 来说没有删除子文件的 Index导致的。毕竟源目录整个目录都没了。被同步的目录还在报目标路径还有文件就很奇怪

  • https://syncthing.net/
  • https://github.com/syncthing/syncthing

后者是核心进程，MPL-2.0
不过我也还没在Linux上用过
