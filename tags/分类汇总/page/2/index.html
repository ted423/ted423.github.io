[软件推荐]有用的小工具们

批量重命名软件类

https://ted423.github.io/Software/%E6%89%B9%E9%87%8F%E9%87%8D%E5%91%BD%E5%90%8D%E8%BD%AF%E4%BB%B6/

Attribute Changer

同File Property Edit一样可以进行文件属性修改，但主要是通过右键菜单，
http://www.petges.lu/home/download/

File Property Edit

文件属性修改,你可以将普通文件改成系统文件，十分适合用来藏东西
http://www.foryoursoft.com/download.htm

搜索类

DocFetcher

开源(EPL)的文件内容搜索，不止支持文本文件，还支持doc以及pdf等文档
不过需要进行索引
最重大的缺点就是不能搜索连续的符号，但有时候需要精准结果的时候，这个软件靠不住
http://docfetcher.sourceforge.net/en/index.html

Everything

搜索文件用，相当出名的文件搜索工具，会通过主文件表中的文件元数据创建卷上每个文件和目录名称的索引，速度相当快，缺点就是只支持文件名，不支持附加条件的搜索，例如文件大小和创建日期
http://www.voidtools.com/

FileSeek

这个软件是专门为了搜索文件内容而生的。没有locate32的小毛病，后面感觉用FileLocator会好些，因为我发现FileLocator添加默认为文本的文件类型会比较方便。我印象里FileSeek最早也是直接添加的，不知道是我记错了还是后面改了。
https://www.fileseek.ca/

FileLocator

跟上面算是同质同类的软件
https://www.mythicsoft.com/filelocatorpro/

locate32

BSD License
个人目前没在用，因为FileSeek或者Docfetcher明显更好用
搜索文件用,虽然相对于Everything来说不是那么快速，但是功能更为强大，可以搜索文件大小、日期等等
就是文件内容搜索是将DBCS与UTF-8分开的，搜索utf-8文本文件的话，需要以utf8:开头。
长期没有更新了
https://sourceforge.net/projects/locate32/

Copyq

GPLv3
用就是了 之前在用Ditto，不过感觉Ditto小bug有点多
https://github.com/hluk/CopyQ

Exeinfo PE

放在Github上的闭源软件,之前有一堆不同域名的主页
https://ted423.github.io/Software/Exeinfo%20PE/

HashCheck

Standard 3-Clause BSD License.
https://github.com/gurnec/HashCheck
因为Robin Keir的hash在特殊字符上有些问题，找替代软件找到的。不过只能在右键属性里查看。如进行文件对比，操作量会略微增加。

MyHash

Robin Keir的hash的完全替代品,可惜虽然在Github上，但没有开源。
https://github.com/drag0n-app/MyHash

KeePassXC

强大的密码管理软件以及密码自动填写工具
https://ted423.bitcron.com/post/software/keepassxc

微软小工具合集

微软推荐的系统工具合集
https://ted423.bitcron.com/post/software/xi-tong-gong-ju/wei-ruan-xiao-gong-ju-ji-he

Registry Workshop

注册表搜索工具,搜索速度快，可以批量操作,不过需要付费
https://hrtsea.com/2133.html
官网：http://www.torchsoft.com/

RightMenuMgr

右键菜单管理，最详尽最专业的，官网已消失，虽然是老软件但在win10下也正常。
https://www.lanzous.com/i39jpri

SpaceSniffer

是一款非常直观的磁盘空间占用查看应用，还支持根据不同类型文件进行过滤显示。后面感觉WizTree更好
http://www.uderzo.it/main_products/space_sniffer/

WizTree

比较而言后者大文件提示会比较直观.百分比占用会比图形更准确，好理解。
前者会边扫图形边变化，前期图形会失真，并且整体速度略慢
据说Wiztree是直接对NTFS文件系统的MFT进行文件分析，不过我测试是对的网络路径(小文件太多估计有影响)
固此，我倾向于用WizTree 完全取代 SpaceSniffer
https://www.diskanalyzer.com/

SUMo

软件升级检测，最神奇的是他会跟据快捷方式来检测软件，还可以自己添加检测文件夹。不单单是从系统的程序和功能里检测，也就是一些绿色软件甚至单exe的小工具也能检测到。
其中扫描就是确定有什么软件以及当前软件版本，检查就是去服务器查最新版本。
其实这个公司还有出一个检测驱动的叫DUMo,不过会有乱码并且Win10自己有升级驱动的机制了。
http://www.kcsoftwares.com/?sumo

UnLock IT

文件解锁，删除删不掉的文件都可以用
各种解锁软件在我上Win10就开始出问题，比如IObit Unlocker,又比如LockHunter然后我找到了UnLock IT，这是一个免费软件,功能强大。
http://emcosoftware.com/unlock-it/download

[软件推荐]图片类相关软件

看图

最近没怎么看漫画就比较尴尬，目前还是Honeyview，能用就行，不过感觉他的缩放算法问题很大

我一度是想抛弃ACDsee的，比如文件名长度问题导致的崩溃，又比如某些gif会花。但偏偏他的数据库让人无法舍弃，比方说按宽高搜索图片之类的，速度相当快，相对来讲XnViewMP搜索类别更细，功能性更强，但是Xnview里的结果是一次性的，一旦点开一张图片结果就会消失。就个人感觉，XnView的规划很乱，设计也是乱七八糟，唯有功能比较多而已。
况且XnView和XnViewMP的使用界面和方式还有很多不一样，感觉怪怪的。
MassiGra的各种按键自定义简直可怕，而且支持读取压缩包里的图片，其实SumatraPDF也支持读取压缩包里的图片
强力推荐下Honeyview，给我一种“就是他了”感觉的软件，比MassiGra简单很多，但又能完全取代的一款软件
因为Honeyview说除安全修补，不再更新，所以想找替代，结果Honeyview 2021还有更新，nomacs一年没更新(现在都在action里下最新版)，FastStone MaxView两年。

  • XnViewMP :http://www.xnview.com/en/xnviewmp/ (QT开发跨平台)
  • Honeyview:http://www.bandisoft.com/honeyview/cn/ (商业软件)
  • nomacs: https://nomacs.org/ (GPLv3)
  • ACDsee Ultimate:-|官网
  • MaxView: http://www.dayanzai.me/maxview.html|官网

看压缩包漫画，首选还是Honeyview，SumatraPDF目前感觉用起来不错，nomacs只支持zip/cbz
但如果看对比图或者比较专用的使用的话Honeyview就不太行，滚动图片会移位

图片管理

Xnview

挂个名

Adobe Bridge CC

http://www.adobe.com/cn/products/bridge.html

这款软件是我找来取代ACDsee的图片管理功能的，毕竟ACDsee收费，并且其不能在资源管理器里打开，也不能用其他图片浏览软件打开图片，Adobe Bridge CC 虽然不像ACDsee那样可以根据图片的宽和高来搜索图片，但可以根据图片的长宽比来分类，缺点是界面有点丑。
值得一提的是Adobe lightroom CC 反而不支持通过图片的长宽比来分类，简直让人难以理解。

digiKam

这是一款根植于 Linux 的 KDE 软件，但是跨平台,可以编辑，就是单纯看图不太行，不过可以调用看图软件。目前我的首选。
不过Linux软件讨厌的一点是一不注意就往源目录下”拉屎”,注意不要用他的回收站，不是指的往Windows的回收站丢。
