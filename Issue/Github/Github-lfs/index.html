Github git-lfs 注意事项

  • 2025-01-10
  • 作者 ted423
  • 748 字
  • 次阅读

其实主要就是 Github 的 lfs 空间只有1G。 但是坑的点在于。

  1. 在达到限制前不会有任何提醒。
  2. 空间使用量藏得很深。

然后就是第二方面，网上所有的教程都是删除本地 lfs 文件的。实际你没法自己删除 Gihub Server 上的 lfs 文件。

目前唯一的处理办法是 https://stackoverflow.com/questions/34579211/how-to-delete-a-file-tracked-by-git-lfs-and-release-the-storage-quota

  1. Open support github ticket https://support.github.com/contact?tags=docs-generic%2Crr-general-technical
  2. Write subject title as "remove git lfs file"
  3. Virtual Agent button should appear below
  4. Click that to open dialog with bot
  5. Click "Remove a file(s) from LFS
  6. Bot responds with guide to get LFS file ID's
  7. AND you need to remove file from history also https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/removing-sensitive-data-from-a-repository
