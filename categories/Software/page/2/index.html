[开源]软件更新管理 - Winget

Winget 打败了所有（
所以地址由 Software Update 改成了 Winget

Winget

开源，MIT，开源地址 https://github.com/microsoft/winget-cli
Winget 永远的神，Win 10 开始自带
目前除了批量升级有点使用上的问题以外，其他做的都挺强

目前可以通过以下方式批量升级(也可以用来备份恢复软件列表)

  1. winget export -o 文件
  2. 删除列表里不需要的文件
  3. winget import 文件

winget 下的升级文件都在 %Temp% 里，一些软件升级时需要关闭其他软件可能会导致升级失败，可以手动运行安装包升级

https://winstall.app/apps

这里可以自己选软件然后生成批量安装的代码

WingetUI

开源 LGPLv2.1

WingetUI 其实不依赖系统的 winget,他自己又带了一个，系统的是appx，然后配置放在软件的同目录下，而 WingetUI 的 winget 的配置是放在 C:\Users\XX\AppData\Local\Microsoft\WinGet — 2023-07-18

https://www.marticliment.com/wingetui/
https://github.com/marticliment/WingetUI
第三方UI，目前没弄懂为啥他获取的会比命令行的要新
不过有忽略模式就行。

首先，下面两个软件主要是替代国产软件管理用的，并且属于一种不成熟的替代品。
如果你想寻找一种更好的体验，那大概是可以放弃看这篇文章。

Sumo

http://www.kcsoftwares.com/?home
夸一夸Sumo，真的很强大，他的软件检测是基于快捷方式，也就是便携版也支持检测。这点已经比360等的强了。
不过不给钱他只是一个检测器而已，甚至连从开发者网站获取更新都不让你用。

这个软件商还有个产品Dumo是升驱动的，不过Win已经大包大揽，再不济还有各类厂商的自用检测。(有点Out of topic)

Chocolatey

Chocolatey 虽然是开源的，但其实际还是一款面向企业的商业产品，并且国内体验不好。Apache License 2.0
像什么微信QQ企业微信，里面是没有的。
免费产品限制了功能，并且只能装到C盘。之前还有安装失败也会计入软件列表的问题

因此这个产品的主要定位是一定程度上处理软件升级的问题。主要处理软件自带升级智障的问题，例如WinSCP(之前有用scoop，这个自己本身也智障，配置放便携版里，结果一更新配置没了)

GUI

choco install chocolateygui

注意的点

  • 这软件在Win7上要更新PowerShell。
  • 只有想装C盘并且不需要安装包配置关联的软件才用Chocolatey直接装，否则建议自行安装，其后用Chocolatey升级
  • 社区维护，因此很多软件安装行为都由社区维护者决定
  • 因为上面一条，可能面临没有维护或者不再维护的情况
  • 网络问题
  • 还可能有升级报错的情况，如OBS-Studio
  • 部分时候需要重装.install https://github.com/chocolatey/choco/issues/1027
  • 他的软件列表和计算机的列表是相互独立的，并且不能单纯只删记录(只能pin)

一些常用命令

  • 列出本地安装软件 choco list --local-only
  • 列出可升级的软件 choco outdated
  • 安装指定等级 choco install virtualbox --version 4.3.26
  • 不检查更新choco pin add -n=googlechrome
  • choco search 名字
  • Proxy Server (不支持socks5,)

然后在金山卫士开始有一堆受众的时候把卫士砍了。

[推荐]Directory Opus - 多标签资源浏览器

https://www.gpsoft.com.au/index.html
不得不说，资源浏览器这块，Linux走在Win的前面。

我基本试了所有多标签资源浏览器， 并没能找到一个完美的资源浏览器。
（有个ExTab还没试，类似Clover，我当时买DO时他刚刚出生，不过免费软件嘛……不开源的情况下等作者累了就没人维护了，特别时只有QQ群和公众号的情况下,Issue都不好管，不过我可以给别的电脑装装玩玩）
DO算是里面让我感觉最好的了。

所以我最终入了正

别的软件的问题有

  • 干扰到平时的正常操作，这点Clover、QTTabBar都存在问题，QTTabBar会影响复制粘贴，Clover在有文件改动的时候会频繁刷新界面，导致没法操作
  • Clover转手后，基本没有很好的维护
  • QTTabBar没办法只在一个窗口显示，有时会打开新的窗口。
  • ExTab拖拽文件卡死(不算大问题，取决于作者的维护力度)
  • Groups会卡死。
  • 没法替代资源管理器，有时候别的软件调用资源管理器会直接打开资源管理器，印象里QTTabBar能做到
  • 维护稳定性不可保证，一些已经长期不更新

DO存在的问题有

  • 我个人喜欢Win原生的界面就好，但DO的开始界面是自己实现的，不过问题不大
  • 一些功能仍然只有Win原生才能做到，如对快速访问的操作，固定到开始菜单，通过集成方式实现功能的能做到，但是自己写资源管理器的软件做不到（这个是Win自身的限制）
  • 设置项过多，不够简练，有些设置项甚至是不必要的。会让人找不到东西。我个人是不喜欢那种啥都有，但啥都不精的软件。例如压缩文件的相关操作，7z操作不如7z,rar操作不如winrar。

并且DO经常更新，平时使用非常稳定,甚至有很多我懒得用的功能，所以挺好的

文件系统加密 否 是
事务 否 是
对象 ID 否 是
卸载数据传输 (ODX) 否 是
短名称 否 是
扩展的属性 否 是
磁盘配额 否 是
可引导 否 是
页面文件支持 否 是
在可移动媒体上受支持 否 是

以下功能仅可用于 REFS

功能 ReFS NTFS
块克隆 是 否
稀疏 VDL 是 否
镜像加速奇偶校验 是（在存储空间直通上） 否
文件级快照 是1 否

其他不说就算了，引导啥的可有可无，毕竟Refs目标是存储用的文件系统。

网络同步盘缺陷记录

优秀

  • dropbox
    同步不够实时(服务器端的改变并不会立刻同步回本地)
    如果你希望他立刻进行一次完全的同步，你需要退出他的客户端再重新打开。
  • Onedrive
    不支持局域网内同步,同步逻辑有缺陷，偶尔会干删一堆文件到回收站再重新下载的事(由于下调存储空间一事，已经不准备继续使用，准备转yandisk看看，15元/月才50G，我就呵呵了)
  • 快盘
    国内做的最好的同步盘，不过服务器实在…
    客户端弹窗简直烦人
    因为有秒传机制，会出现上传的某些文件永远无法下载（你也没法重传，因为秒传了）
    因为上一条，是否使用自己斟酌

可用

  • mega
    支持创建同名文件(夹)，不利于同步，同步依据文件修改时间，在修改时间不变改变文件的情况下会去下载源文件覆盖修改后的文件。
    在其他电脑上直接在网页端上传文件同步回本地后会再次进行上传（可能跟装了浏览器扩展有关），然后创建出同名文件夹(我这里甚至出现了重启次机子就会重新上传的状况)
  • Yandex
    同步时会导致同步文件夹内的文件写入慢，影响其他读取该文件夹的软件运行
    不支持回复文件历史
    * 新浪微盘同步盘
    同步时暴力拒绝其他修改，你可以把他理解为一个手动同步工具…
    需要修改时你就暂停他的同步吧。
    * 酷盘(已死)

不推荐使用

  • Google Drive
    这同步简直败坏谷歌的名声（虽说谷歌做的所有软件一直不是我喜欢的，因为感觉太偏执了）
    支持创建同名文件(夹)，不会进行匹配，所以多端同步做的简直shit,你只能让他自己去同步。如果你自己往里放文件协助同步你会发现服务端建立了一堆同名文件或文件夹，然后你会在另一端看到一堆wrong,逗我。
  • 360同步
    只支持单任务，在上传时居然会不去判断网盘内的修改，然后就出现了完全重复的两个文件。基本只是个玩具，根本算不上同步盘。如果你希望他把你的资料弄得乱七八糟，你可以使用他。
  • 百度云同步
    服务器相当不稳定，一般的同步问题都是服务器导致的，甚至有时明明同步未完成，网盘也会判断同步已完成。百度对部分视频进行了破坏性修改，突破了网盘的底线，因此不建议使用这个网盘存储自己的资料
  • 微云
    我当时用的时候，微云是文件名同步，即便改变文件内容，微云也不会去重新同步
