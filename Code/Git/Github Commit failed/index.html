[Git]处理Github的Commit failed:Failed to create a new commit

প্রকাশিত হয়েছে 2015-01-01 শ্রেনী Code

2015.02.15确认没再出现类似错误，大概Github的Windows客户端更新了

好吧，被坑了好多次，之前都是被搞成了删除项目重建……
现在终于搞懂了，是.git文件夹里有个index.lock的问题…..
只要删掉就好了。大概在创建commit的时候就会自动生成，但此时如果又更改了项目文件的话，
就需要重新生成commit,但由于之前已经有了index.lock（但没生成commit），而git不允许index.lock的覆盖，所以就卡住了…
