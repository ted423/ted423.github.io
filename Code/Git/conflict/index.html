[Git]conflicts 处理

نُشر في 2017-03-10 في Code

rebase 可能会造成冲突
多人同时编辑也会造成conflict

需要将所有冲突的commit纳入rebase，此时rebase会提示conflict并且暂停

其实此时你的本地文件会变成冲突时的文件，你要做的就是编辑该文件将冲突处
也就是Github使用

括起来的冲突内容

中只保留你需要内容，使用 git add (文件)

随后你只要git rebase --continue结束rebase或者开始处理下一个conflict即可
