WordPress修改固定链接之后,无法访问的解决办法

نُشر في 2016-05-04 في Code

原文链接:http://www.inlojv.com/wordpress-modify-fixed-link.html

主要是方法二所说的问题，也就是apache里mod_rewrite 模块没有启用。

修改apache里面的配置文件httpd.conf

主要修改两个地方

  1. 开启apache中的rewrite模块，在httpd.conf 中找到
    #LoadModule rewrite_module modules/mod_rewrite.so
    把前面的#号去掉，变成
    LoadModule rewrite_module modules/mod_rewrite.so

  2. 开启AllowOverride ，怎么做？ 也是在httpd.conf 中找到AllowOverride None将它改成AllowOverride All
    特别注意的是：AllowOverride可能会有好几处，只需要修改 <Directory /> 和 你站点所在的根目录的一处
    如<Directory “F:/…… /www/htdocs”> 即可
    如果不懂，索性将这几处全部改了 。全部改好，必须保存 并且重启apache 。
    问题即可解决。
