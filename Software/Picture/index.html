看图

最近没怎么看漫画就比较尴尬，目前还是Honeyview，能用就行，不过感觉他的缩放算法问题很大

我是想抛弃ACDsee的，比如文件名长度问题导致的崩溃，又比如某些gif会花。但偏偏他的数据库让人无法舍弃，比方说按宽高搜索图片之类的，速度相当快，相对来讲XnViewMP搜索类别更细，功能性更强，但是Xnview里的结果是一次性的，一旦点开一张图片结果就会消失。就个人感觉，XnView的规划很乱，设计也是乱七八糟，唯有功能比较多而已。
况且XnView和XnViewMP的使用界面和方式还有很多不一样，感觉怪怪的。

MassiGra的各种按键自定义简直可怕，而且支持读取压缩包里的图片，其实SumatraPDF也支持读取压缩包里的图片
强力推荐下Honeyview，给我一种“就是他了”感觉的软件，比MassiGra简单很多，但又能完全取代的一款软件
因为Honeyview说除安全修补，不再更新，所以想找替代，结果Honeyview 2024还有更新，nomacs一年没更新(现在都在 Github pre-release 里下最新版)，FastStone MaxView两年。

nomacs 的一个问题 https://github.com/nomacs/nomacs/issues/237 --2024-07-27

  • XnViewMP :http://www.xnview.com/en/xnviewmp/ (QT开发跨平台)
  • Honeyview:http://www.bandisoft.com/honeyview/cn/ (商业软件)
  • nomacs: https://nomacs.org/ (GPLv3)
  • ACDsee Ultimate:-|官网
  • MaxView: http://www.dayanzai.me/maxview.html|官网

看压缩包漫画，首选还是Honeyview，SumatraPDF目前感觉用起来不错，nomacs只支持zip/cbz
但如果看对比图或者比较专用的使用的话Honeyview就不太行，滚动图片会移位

图片管理

Xnview

挂个名

Adobe Bridge CC

http://www.adobe.com/cn/products/bridge.html

这款软件是我找来取代ACDsee的图片管理功能的，毕竟ACDsee收费，并且其不能在资源管理器里打开，也不能用其他图片浏览软件打开图片，Adobe Bridge CC 虽然不像 ACDsee 那样可以根据图片的宽和高来搜索图片，但可以根据图片的长宽比来分类，缺点是界面有点丑。
值得一提的是Adobe lightroom CC 反而不支持通过图片的长宽比来分类，简直让人难以理解。

digiKam

这是一款根植于 Linux 的 KDE 软件，但是跨平台,可以编辑，就是单纯看图不太行，不过可以调用看图软件。目前我的首选。
不过 Linux 软件讨厌的一点是一不注意就往源目录下"拉屎",注意不要用他的回收站，不是指的往 Windows 的回收站丢。
最后卸了，因为"拉屎"这个问题。

工具

  • GIMP:http://www.gimp.org/(取代photoshop用的，Linux原生的跨平台的软件)
  • ShareX: https://getsharex.com/ | Github | 介绍
  • Greenshot:http://getgreenshot.org/|Github|官网|Bitbucket (开源的截屏工具，GPLv3)诈尸维护，但是没 Release
  • GifCam:http://www.dayanzai.me/gifcam.html(录屏工具)
  • LICEcap:http://www.cockos.com/licecap/(录屏工具，并且开源免费，GPLv2)
  • Inkscape: https://inkscape.org/(SVG编辑工具,GPLv2)
  • ImageMagick: https://imagemagick.org/index.php (开源，但协议是自创的)之前XnViewMP转换遇到问题，找的，挺强大的，不过Xnview遇到的问题应该是那些图片转化本身的bug,ImageMagick在同样的转换下也会出问题
  • oxipng png无损压缩优化软件，会生成最小的png，MIT协议
  • caesium 开源的图片批量压缩工具(GPLv3)，主要 Xnview 商业使用需要付费，这个功能上是不如 Xnview 的
