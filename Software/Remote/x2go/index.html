[开源]x2go - 远程连接

  • 2025-01-10
  • 作者 ted423
  • 728 字
  • 次阅读

不推荐使用，最后又滚回去使用NoMachine了，主要是目前维护不活跃。

然后发现NoMachine有别的问题

下面记录了一些早期测试的记录，不过当时有点菜，很多操作没有仔细去确认，例如 Linux 普通台式机会在没有外接显示器的情况下，把 GPU 置入省点模式，此时画面显示会非常卡，我目前都是给台式机上显卡欺骗器的。

网址

https://wiki.x2go.org/doku.php/start

闲着无聊试新的远程方式，看看有没合心意的，之前试过QNAP的spice，结果默认剪贴板也是没得。(QNAP 系统渣，还瞎魔改)

看到x2go用NX协议就也想试试，毕竟开源。

首先是跟nomachine不兼容。

当然这是小事。

x2go支持的桌面是有限的，像什么KDE5不支持，所以你需要使用xfce等在支持列表上的桌面。

使用桌面登录会跟你显示器登录是不一样。

当然还能用X11

这里引入一个小故事

当我看到x2go在x11可以选择桌面的时候我是很激动的(x11)(

毕竟这意味着我可以弄一个别人看不到的桌面。

然后我闲着无聊重启了下我就傻逼了。

我连不进去了，因为 Debian 同学登陆么的桌面，于是我跑到电脑前，登录进了系统，然后再跑回位置上连。(直接使用xfce这些桌面没这个问题。)

遇到的问题

  1. 与显示器同步的画面必须要到电脑前登录，否则x2go的x11 foward不工作。(这点 NoMachine 默认只支持同步显示，并且无需现场登录)
  2. 利用x2go创建的远程画面会卡死。只能强关，关了之后原画面显示内容丢失，甚至我重启开那一会不动，桌面就死了。当然这可能是Mint的锅，不过谁知道呢(我又仔细看了下，我装的是xubuntu,那没事了)
