上 Nextcloud 主要是我对于 Dropbox 的一些不满化为了行动。

  1. Dropbox 长期在右键菜单里拉屎
  2. 很多次同步卡住(后面发现文件有些奇怪的状态)
  3. 经常被它锁定文件打断操作
  4. 长期接U盘/移动硬盘弹窗拉屎
  5. 会因为个别文件占用阻塞整个同步操作

Nextcloud 那个忽略文件名还是很有用的，毕竟有些文件夹，例如 node_modules 没啥用还经常造成问题。

近期大概是欧盟资金支持(?), Nextcloud 肉眼可见的靠谱了起来

问题

但 Nextcloud 的问题也不少，甚至更可怕

已解决

  1. 超长的同步检查时间。甚至会对设置了忽略的文件夹进行检查。并且是肉眼可见的一个个检查
  2. Release 版本可能长期有各种问题，最新的客户端(3.14.*)很喜欢崩溃，崩溃这个问题比较严重，毕竟同步盘肯定是要长期同步的，Linux 的还好(长期3.11版本有关)
  3. 同步机制问题，其他端删除的文件会被新开机的电脑重新同步回去
  4. 无法开始同步,他会在统计文件数量的时候崩溃，然后无法开始同步 -2025-01-09update
  5. https://github.com/nextcloud/desktop/issues/7594
  6. 还有文件丢失时间标签无法同步的 issue 关了 https://github.com/nextcloud/desktop/issues/4378 未作确认
  7. 本地产生了4GiB的log -2025-01-09

不确定状态以及影响

  1. https://help.nextcloud.com/t/unable-to-upload-sync-htaccess-files/47523/19
  2. 需要管理员权限来删除快捷方式 -2025-01-09

还有影响的

  1. 目前同步时间较慢，不过这个有可能也要看 Server 性能
  2. Dropbox 支持 LAN 同步，它不支持
  3. https://github.com/nextcloud-snap/nextcloud-snap/issues/1031#issuecomment-1670204085

整体来说，东西还是 Dropbox 比较保险，带 node_modules 的才放 Nextcloud 里
目前确认 Nextcloud 同步机制问题很大。
